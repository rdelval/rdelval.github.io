Renan DelValle https://cs.binghamton.edu/~rdelval1 PhD Candidate at SUNY Binghamton Sat, 23 Sep 2017 18:12:34 +0000 en-US hourly 1 https://wordpress.org/?v=4.8.1 Installing Linux when there is no BIOS support for booting from USB https://cs.binghamton.edu/~rdelval1/installing-linux-when-there-is-no-bios-support-for-booting-from-usb/ Thu, 02 Jun 2016 02:56:09 +0000 http://cs.binghamton.edu/~rdelval1/?p=54 Many old computers (pre Vista era) have no support for booting from a USB drive.

This became an issue the moment Linux distros were only made to be installed from a DVD drive. That’s because most of the computers that don’t have support for booting from a USB also don’t have DVD drives.

This meant that I was limited to installing distros that fit on a CD, such as Ubuntu minimal. This is fine for most as you can choose to install your favorite DE during the menu selection in the install process.

In this case, most of my lab has become very familiar with Linux Mint which we use due to its similarity to the “XP” interface so many are used to. In short, it’s much easier for new students to work with the system (not against it!) and get to work.

However, since around Linux Mint 14, the install image doesn’t fit on a DVD. A little searching brought me to the Plop Boot Manager. It’s a cool little project that does away with any booting inadequacies the BIOS has. It boots to a simple interface which then allows you to boot from a USB drive.

Unfortunately for me, it didn’t work at first. The drive containing Linux Mint 17.3 XFCE refused to boot. By chance I happened to enter the settings and tweaked them to force USB 1.1 in Mode 1. This let me finally boot into the live environment and install from the USB.
